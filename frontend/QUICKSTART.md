# 🚀 DB-Doctor 前端快速启动指南

> **版本**: v3.0.0
> **更新日期**: 2026-02-04

---

## 🎯 快速开始

### 1. 安装依赖
```bash
cd frontend
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

服务器将在 **http://localhost:3000** 启动

### 3. 访问页面

#### 主页面
- **仪表盘**: http://localhost:3000/
- **慢查询诊疗**: http://localhost:3000/diagnostics
- **设置中心**: http://localhost:3000/settings

#### 设置中心子页面
访问 http://localhost:3000/settings 后：

**实例管理**：
- 点击"实例管理"卡片
- 切换"数据库实例"或"AI 服务实例"标签

**基础配置**：
- 点击"基础配置"卡片（开发中）

**监控与通知**：
- 点击"监控与通知"卡片（开发中）

## ✨ 核心功能演示

### 数据库实例管理

#### 步骤 1: 新增实例
1. 访问 http://localhost:3000/settings
2. 点击"实例管理"卡片
3. 确保在"数据库实例"标签
4. 点击右上角"新增数据库实例"按钮

#### 步骤 2: 填写表单
```
实例名称: 生产环境主库
主机地址: 192.168.1.100
端口: 3306
用户名: root
密码: your_password
环境类型: 生产环境
```

#### 步骤 3: 测试连接
1. 点击"测试连接"按钮
2. 等待测试完成（约1-2秒）
3. 查看测试结果：
   - ✅ 成功：显示绿色提示和延迟
   - ❌ 失败：显示红色提示和错误信息

#### 步骤 4: 保存实例
1. 测试成功后，"保存实例"按钮变为可用
2. 点击"保存实例"
3. 实例出现在列表中

#### 步骤 5: 管理实例
- **编辑**: 点击卡片上的"编辑"按钮
- **删除**: 点击卡片上的"删除"按钮
- **启用/禁用**: 切换右上角开关
- **搜索**: 使用顶部搜索框过滤实例

### AI 服务实例管理

#### 步骤 1: 选择 Agent
1. 在"实例管理"中切换到"AI 服务实例"标签
2. 看到三个 Agent 卡片：
   - 🩺 主治医生（紫色）
   - 🧠 推理专家（粉色）
   - 💻 编码专家（蓝色）

#### 步骤 2: 配置 Agent
1. 点击任意 Agent 卡片的"配置"按钮
2. 在右侧抽屉中选择提供商：
   - OpenAI
   - Ollama
   - DeepSeek
   - Anthropic
   - Azure OpenAI

#### 步骤 3: 填写配置
```
提供商: OpenAI
Base URL: https://api.openai.com/v1
API Key: sk-your-api-key
模型: gpt-4-turbo
Temperature: 0.7
Max Tokens: 2000
```

#### 步骤 4: 测试和保存
1. 点击"测试连接"
2. 测试成功后点击"保存配置"
3. Agent 卡片更新为"已配置"状态

## 🎨 UI 特性展示

- **深色渐变**: 从 `#1e293b` 到 `#0f172a`

## 🐛 常见问题

### Q1: 页面显示空白或报错
**A**:
1. 检查浏览器控制台（F12）是否有错误
2. 确保已运行 `npm install`
3. 尝试清除缓存：`Ctrl + Shift + R`

### Q2: 样式显示不正确
**A**:
1. 确认 `main.ts` 中已引入 `@/styles/global.css`
2. 检查浏览器是否支持 CSS 变量
3. 清除浏览器缓存

### Q3: 组件导入错误
**A**:
1. 检查文件路径是否正确
2. 确认组件文件是否存在
3. 查看控制台的具体错误信息

### Q4: API 请求失败
**A**:
- 这是正常的！当前使用模拟数据
- 后端 API 还未实现
- 不影响前端 UI 和交互的测试
