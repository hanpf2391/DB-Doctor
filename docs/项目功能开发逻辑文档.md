## 一、项目架构概览

### 1.1 技术栈

**前端**
- Vue 3 + TypeScript
- Vite 5.x（构建工具）
- Element Plus（UI 组件库）
- ECharts（数据可视化）
- Axios（HTTP 客户端）
- Vue Router 4.x（路由管理）

**后端**
- Java 17
- Spring Boot 3.x
- Spring Data JPA
- H2 Database（内嵌数据库）
- LangChain4j（AI 集成框架）
- JavaMail Sender（邮件服务）

### 2.2 路由守卫机制

**认证流程：**

```typescript
// 1. 恢复登录状态（首次访问时）
if (!authStore.isAuthenticated && !authStore.username) {
  authStore.restoreSession() // 从 localStorage 恢复
}

// 2. 需要认证的页面
if (to.meta.requiresAuth !== false) {
  if (authStore.isAuthenticated) {
    next() // 已登录，放行
  } else {
    next({
      path: '/login',
      query: { redirect: to.fullPath } // 保存原始路径
    })
  }
}

// 3. 已登录用户访问登录页，跳转到首页
if (to.path === '/login' && authStore.isAuthenticated) {
  next('/')
}
```

**Token 管理：**
- 存储位置：`localStorage.getItem('dbdoctor_token')`
- 传递方式：HTTP Header `Authorization: Bearer ${token}`
- 过期处理：401 响应时自动清除 token 并跳转登录页

## 三、核心功能模块详解

### 3.1 仪表盘（Dashboard）

#### 3.1.1 功能概述

仪表盘是系统的首页，提供慢查询数据的全局概览。

**文件路径：** `frontend/src/views/Dashboard/Dashboard.vue`

#### 3.1.2 核心功能

**1. 统计卡片（4个）**

| 卡片名称 | 数据来源 | 说明 |
|---------|---------|------|
| 模板总数 | `stats.templateTotal` | 去重后的慢查询模板数量 |
| SQL 样本总数 | `stats.sqlTotal` | 所有慢查询样本的总数 |
| 高危 SQL 数 | `stats.highRiskCount` | 严重程度为 CRITICAL 的数量 |
| 待分析任务 | `stats.pendingTasks` | 状态为 PENDING 的任务数 |

**2. 趋势图（24小时慢查询趋势）**

- **数据来源：** `/api/dashboard/trend?date=YYYY-MM-DD`
- **图表类型：** 折线图 + 面积图
- **X 轴：** 小时（0-23）
- **Y 轴：** 慢查询数量
- **交互：**
  - 今日/昨日快捷按钮
  - 日期选择器
  - 点击刷新按钮

**3. Top 5 慢查询榜单**

- **数据来源：** `/api/dashboard/top?limit=5`
- **排序规则：** 按 `maxQueryTime` 降序
- **图表类型：** 横向柱状图
- **颜色编码：**
  - 红色 (#f56c6c)：严重
  - 橙色 (#e6a23c)：警告
  - 灰色 (#909399)：注意
  - 绿色 (#67c23a)：正常

**4. 快捷操作**

- 查看所有报告 → 跳转到 `/reports`
- 配置中心 → 跳转到 `/settings`
- 刷新数据 → 并发加载所有数据

#### 3.1.3 API 对接

```typescript
// 1. 获取今日概览
GET /api/dashboard/today-overview
Response: {
  templateTotal: number,
  sqlTotal: number,
  todayTotal: number,
  highRiskCount: number,
  avgQueryTime: number,
  pendingTasks: number,
  date: string
}

// 2. 获取趋势数据
GET /api/dashboard/trend?date=2026-02-10
Response: {
  date: string,
  hours: number[],    // [0, 1, 2, ..., 23]
  counts: number[]    // 每小时的慢查询数量
}

// 3. 获取 Top N
GET /api/dashboard/top?limit=5
Response: {
  records: [{
    id: number,
    fingerprint: string,
    sqlTemplate: string,
    dbName: string,
    tableName: string,
    maxQueryTime: number,
    severityLevel: string
  }]
}
```

#### 3.1.4 数据加载逻辑

```typescript
onMounted(() => {
  refreshAll() // 并发加载三个数据源
})

async function refreshAll() {
  await Promise.all([
    loadStats(),   // 统计卡片
    loadTrend(),   // 趋势图
    loadTop()      // Top 5
  ])
}
```

### 3.3 AI 监控

#### 3.3.1 功能概述

AI 监控模块提供 AI 调用链路追踪、成本分析和性能监控。

**目录结构：**
```
frontend/src/views/AiMonitor/
├── index.vue           # 监控总览
├── InvocationLog.vue   # 调用日志
├── AnalysisTraceList.vue # 链路追踪
└── CostAnalysis.vue    # 成本分析
```

#### 3.3.2 监控总览（`index.vue`）

**核心指标卡片：**

| 指标名称 | 说明 |
|---------|------|
| 总调用次数 | AI 模型的总调用次数 |
| 成功调用次数 | 成功返回结果的次数 |
| 失败调用次数 | 发生错误的次数 |
| 平均响应时间 | AI 模型的平均响应时间 |
| 总 Token 消耗 | 输入 + 输出 Token 总数 |
| 预估成本 | 基于 Token 估算的成本 |

**图表展示：**

- 调用趋势图（折线图）
- Agent 调用分布（饼图）
- Token 消耗趋势（面积图）

#### 3.3.3 调用日志（`InvocationLog.vue`）

**列表字段：**

- Trace ID（链路追踪 ID）
- Agent 名称（DIAGNOSIS/REASONING/CODING）
- 模型名称（GPT-4/Claude 等）
- Prompt 长度
- Response 长度
- Token 消耗
- 耗时
- 状态（SUCCESS/FAILED）
- 时间戳

**搜索过滤：**

- 按 Trace ID 搜索
- 按 Agent 类型过滤
- 按状态过滤
- 按时间范围过滤

#### 3.3.4 链路追踪（`AnalysisTraceList.vue`）

**功能：**

- 展示完整的慢查询分析链路
- 按指纹分组展示所有 AI 调用
- 显示每个 Agent 的调用顺序和结果
- 支持 Trace ID 跳转查看详情

**链路展示：**

```
指纹: abc123...
├─ [1] 主治医生初步诊断
│  ├─ Agent: DIAGNOSIS
│  ├─ Model: gpt-4
│  ├─ Token: 1234 / 5678
│  ├─ 耗时: 3.2s
│  └─ 状态: SUCCESS
├─ [2] 推理专家深度推理
│  ├─ Agent: REASONING
│  ├─ Model: gpt-4
│  ├─ Token: 2345 / 6789
│  ├─ 耗时: 4.5s
│  └─ 状态: SUCCESS
└─ [3] 编码专家生成优化代码
   ├─ Agent: CODING
   ├─ Model: gpt-4
   ├─ Token: 3456 / 7890
   ├─ 耗时: 2.8s
   └─ 状态: SUCCESS
```

#### 3.3.5 成本分析（`CostAnalysis.vue`）

**成本统计维度：**

- 按 Agent 类型统计
- 按模型名称统计
- 按日期统计
- 按数据库/表统计

**图表展示：**

- 成本趋势图（折线图）
- Agent 成本对比（柱状图）
- 模型成本占比（饼图）

```java
@Scheduled(cron = "${db-doctor.notify.batch-cron:0 0 * * * ?}")
// 默认：每小时执行一次
public void batchSendNotifications() {
    // 1. 计算时间窗口（过去 1 小时）
    LocalDateTime windowEnd = LocalDateTime.now();
    LocalDateTime windowStart = windowEnd.minusHours(1);

    // 2. 从通知队列查询 PENDING 状态的记录
    List<NotificationQueue> pendingQueues = notificationQueueRepo
        .findByStatusAndAnalyzedTimeBetween(
            NotificationStatus.PENDING,
            windowStart,
            windowEnd
        );

    // 3. 按 SQL 指纹聚合
    Map<String, List<NotificationQueue>> groupedByFingerprint =
        pendingQueues.stream().collect(Collectors.groupingBy(
            NotificationQueue::getSqlFingerprint
        ));

    // 4. 构建批次报告
    NotificationBatchReport report = buildBatchReportFromQueue(...);

    // 5. 发送批量通知（最多重试 3 次）
    for (int retry = 0; retry < 3; retry++) {
        if (notifyService.sendBatchNotification(report)) {
            break; // 成功，退出重试
        }
        Thread.sleep(5000); // 等待 5 秒后重试
    }

    // 6. 保存到历史表
    saveNotificationHistory(pendingQueues, batchId, sentTime);

    // 7. 删除队列记录
    notificationQueueRepo.deleteAll(pendingQueues);
}
```

### 4.6 智能通知策略

#### 4.6.1 目的

避免通知轰炸，只在必要时发送通知。

#### 4.6.2 通知判断条件

```java
// SlowQueryTemplate.shouldNotify()
public boolean shouldNotify(
    int coolDownHours,          // 冷却时间（小时）
    double degradationMultiplier, // 性能下降倍数
    double currentAvgTime        // 当前平均耗时
) {
    // 1. 检查冷却时间
    if (lastNotifiedTime != null) {
        LocalDateTime coolDownDeadline = lastNotifiedTime.plusHours(coolDownHours);
        if (LocalDateTime.now().isBefore(coolDownDeadline)) {
            log.debug("在冷却期内，跳过通知");
            return false;
        }
    }

    // 2. 检查性能是否下降
    if (lastNotifiedTime != null && lastNotifiedAvgTime != null) {
        double degradation = currentAvgTime / lastNotifiedAvgTime;
        if (degradation < degradationMultiplier) {
            log.debug("性能未明显下降，跳过通知");
            return false;
        }
    }

    return true;
}
```

**配置示例：**

```yaml

  notify:
    cool-down-hours: 24         # 同一 SQL 24 小时内只通知一次
    degradation-multiplier: 1.5 # 性能下降 50% 才再次通知
    severity-threshold: 1.0     # 耗时超过 1 秒才通知
```

    sql = sql.replaceAll("\\b1[3-9]\\d{9}\\b", "1*******");

    // 2. 替换身份证号
