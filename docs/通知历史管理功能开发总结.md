- 新增方法: `insertNotificationQueue()` - 插入通知队列

✅ **NotificationScheduler（已修改）**
- 文件: `src/main/java/com/dbdoctor/service/NotificationScheduler.java`
- 核心改动:
  1. 从 `notification_queue` 读取待发送记录（事件驱动）
  2. 生成批次ID（UUID）
  3. 按SQL指纹聚合（支持时间线）
  4. 立即重试机制（失败后立即重试3次，每次间隔5秒）
  5. 发送成功后保存到 `notification_history` 表
  6. 发送失败后也保存失败记录到 `notification_history` 表
  7. 删除队列记录（队列模式）

✅ **NotificationHistoryService（新增）**

## 二、核心特性

### 2.1 事件驱动架构

```
分析完成 → 插入队列 → 定时任务读取 → 聚合时间线 → 发送邮件 → 保存历史 → 删除队列
```

**优势**:
- ✅ 解决状态覆盖问题
- ✅ 保留完整时间线
- ✅ 队列模式，不累积数据

### 2.2 立即重试机制

```
发送失败 → 5秒后重试 → 再失败 → 5秒后重试 → 再失败 → 5秒后重试 → 仍失败 → 保存失败历史
```

**特点**:
- ✅ 立即重试（不等下次定时任务）
- ✅ 写死3次（不需要配置）
- ✅ 总耗时不超过15秒
- ✅ 快速失败，不占用资源

### 2.3 完整的历史查询

- ✅ 按时间范围查询
- ✅ 按批次ID查询
- ✅ 按SQL指纹查询时间线
- ✅ 统计信息查询

## 三、数据流程图

### 3.1 完整流程

```
┌─────────────────────────────────────────┐
│ 1. 分析完成              │
│    ↓                                     │
│  更新 slow_query_template 表              │
│    ↓                                     │
│  插入 notification_queue 队列             │
│  status = PENDING, retryCount = 0        │
└─────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│ 2. 定时任务执行（每小时）                  │
│    ↓                                     │
│  从 notification_queue 读取 PENDING 记录  │
│    ↓                                     │
│  生成批次ID (UUID)                        │
│    ↓                                     │
│  按SQL指纹聚合（支持时间线）               │
│    ↓                                     │
│  发送邮件（立即重试，最多3次）              │
│    ↓                                     │
├─────────────────┬───────────────────────┤
│ 发送成功          │ 发送失败               │
│ ↓                │ ↓                     │
│ 保存到           │ 保存到                │
│ notification_   │ notification_         │
│ history (SENT)  │ history (FAILED)      │
│ ↓                │ ↓                     │
│ 删除队列记录 ✅   │ 删除队列记录 ❌        │
└─────────────────┴───────────────────────┘
```

### 3.2 时间线示例

```
某个SQL的多次分析记录：

10:05 🔴 CRITICAL - 严重锁等待，平均耗时30秒
  ↓ 插入 notification_queue
  ↓ 定时任务发送邮件
  ↓ 保存到 notification_history

10:20 ⚠️  WARNING - 锁等待改善，平均耗时5秒
  ↓ 插入 notification_queue
  ↓ 定时任务发送邮件
  ↓ 保存到 notification_history

10:35 ✅ NORMAL   - 性能正常，平均耗时1秒
  ↓ 插入 notification_queue
  ↓ 定时任务发送邮件
  ↓ 保存到 notification_history

前端查询时间线：
  GET /api/notifications/timeline?fingerprint=abc123
  返回: CRITICAL → WARNING → NORMAL
```
