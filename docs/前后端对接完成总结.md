## 三、API对接说明

### 3.1 API路径对应关系

| 前端API函数 | 后端接口路径 | 说明 |
|------------|-------------|------|
| `getHistory()` | `GET /api/notifications/history` | 查询历史列表 |
| `getBatchDetail()` | `GET /api/notifications/history/batch/{batchId}` | 查询批次详情 |
| `getTimeline()` | `GET /api/notifications/timeline` | 查询时间线 |
| `getStatistics()` | `GET /api/notifications/statistics` | 统计信息 |
| `getBatchFingerprints()` | `GET /api/notifications/history/batch/{batchId}/fingerprints` | 查询批次指纹列表 |

### 3.2 请求参数对应

**查询历史列表**:
```typescript
// 前端
notificationHistoryApi.getHistory({
  page: 0,
  size: 20,
  startDate: '2026-02-01',
  endDate: '2026-02-08',
  status: 'SENT'
})

// 后端接收
@GetMapping("/history")
public Map<String, Object> queryHistoryList(
    @RequestParam(defaultValue = "0") Integer page,
    @RequestParam(defaultValue = "20") Integer size,
    @RequestParam(required = false) LocalDate startDate,
    @RequestParam(required = false) LocalDate endDate,
    @RequestParam(required = false) String status
)
```

**查询时间线**:
```typescript
// 前端
notificationHistoryApi.getTimeline({
  fingerprint: 'abc123...',
  page: 0,
  size: 10
})

// 后端接收
@GetMapping("/timeline")
public Map<String, Object> queryTimeline(
    @RequestParam String fingerprint,
    @RequestParam(defaultValue = "0") Integer page,
    @RequestParam(defaultValue = "20") Integer size
)
```

## 四、页面使用流程

### 4.1 查看通知历史

1. 打开监控中心 → 通知历史
2. 页面自动加载最近7天的数据
3. 顶部显示统计卡片（总数、成功、失败、成功率）
4. 表格显示所有通知记录

### 4.2 查看详情

1. 点击表格中的"详情"按钮
2. 弹出详情对话框
3. 显示完整信息：批次ID、SQL指纹、AI报告等

### 4.3 查看时间线

1. 点击表格中的"时间线"按钮
2. 弹出时间线对话框
3. 显示该SQL的历史分析记录（按时间倒序）
4. 可以看到严重程度的变化过程

### 4.4 查看统计信息

1. 点击顶部的"统计信息"按钮
2. 弹出统计信息对话框
3. 显示总数、成功数、失败数
4. 按状态和严重程度分组统计

## 九、常见问题

### Q1: 前端页面空白？

**A**: 检查后端是否启动，API是否能正常访问。

```bash
# 测试API
curl http://localhost:8080/api/notifications/history
```

### Q2: 统计信息不显示？

**A**: 统计信息需要选择日期范围后才会显示。

### Q3: 时间线没有数据？

**A**: 时间线需要该SQL有多次分析记录才会显示。新系统可能还没有历史数据。

### Q4: 页面显示"加载失败"？

**A**: 检查后端日志，可能是：
1. 数据库表未创建
2. API路径错误
3. 跨域问题
